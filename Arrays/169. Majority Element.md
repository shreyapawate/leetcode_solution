class Solution {
    public int majorityElement(int[] nums) {
        int count=0;
        int candidate=0;
        for(int i =0;i<nums.length;i++){
            if(count==0){
                candidate=nums[i];
            }
            if(nums[i]==candidate){
                count++;
            }else{
                count--;
            }
        }
        return candidate;
        
}
}
# 169. Majority Element

## Pattern
Boyer-Moore Voting Algorithm

## Recognition
- One element appears more than ⌊n/2⌋ times.
- Need O(n) time and O(1) space.
- Looking for the most frequent element without HashMap.

## Core Idea
Treat majority element as a "vote winner".

- Same element → +1 vote.
- Different element → -1 vote.
- Majority element survives all cancellations because it occurs more than n/2 times.

## Algorithm
1. Maintain candidate and count.
2. If count becomes 0, choose current element as candidate.
3. If current element == candidate, count++.
4. Else count--.
5. Return candidate.

## Complexity
Time: O(n)
Space: O(1)

## Template Insight
Whenever:
- Frequency > n/2 is guaranteed.
- O(1) extra space is expected.

Think Boyer-Moore immediately.

## Common Mistakes
- Using HashMap when O(1) space solution exists.
- Forgetting why cancellation works.
- Applying Boyer-Moore when majority element is NOT guaranteed (verification pass may be needed).
