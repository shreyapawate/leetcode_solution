# 287. Find the Duplicate Number

## Pattern
Floyd's Cycle Detection (Tortoise & Hare)

## Recognition
- Array size = n + 1.
- Values are in range [1, n].
- Exactly one duplicate.
- Cannot modify array.
- O(1) extra space required.

## Core Idea
Treat the array as a linked list.

- Index → Node
- nums[index] → Next pointer

Since one value is duplicated, two indices point to the same node, creating a cycle.

Use Floyd's algorithm:
1. Find the meeting point inside the cycle.
2. Reset one pointer to the start.
3. Move both one step at a time.
4. Their meeting point is the duplicate number.

## Algorithm
1. Initialize slow and fast at `nums[0]`.
2. Move:
   - slow = nums[slow]
   - fast = nums[nums[fast]]
3. Continue until they meet.
4. Reset one pointer to `nums[0]`.
5. Move both one step at a time.
6. The meeting point is the duplicate.

## Pseudocode

```java
slow = nums[0]
fast = nums[0]

// Phase 1: Find meeting point
do:
    slow = nums[slow]
    fast = nums[nums[fast]]
while(slow != fast)

// Phase 2: Find cycle entry
slow = nums[0]

while(slow != fast):
    slow = nums[slow]
    fast = nums[fast]

return slow
```

## Complexity

Time: O(n)

Space: O(1)
