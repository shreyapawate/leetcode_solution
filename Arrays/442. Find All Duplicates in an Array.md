class Solution {
    public List<Integer> findDuplicates(int[] nums) {
        List<Integer> arr= new ArrayList<>();
        Arrays.sort(nums);
       for(int i =0;i<nums.length-1;i++){
        if(nums[i]==nums[i+1]){
             arr.add(nums[i]);
            
        }
        
    }
    return arr;
    }
    
}
# 442. Find All Duplicates in an Array

## Pattern
Index Marking (Cyclic Sort Concept)

## Recognition
- Numbers are in range [1, n].
- Need O(n) time.
- Extra space not allowed.
- Need to find duplicates.

## Core Idea
Use the value as an index.

For a number x:
- Visit index x - 1.
- If nums[x - 1] is already negative, x is a duplicate.
- Otherwise, make nums[x - 1] negative to mark it as visited.

## Algorithm
1. Traverse array.
2. Compute index = abs(nums[i]) - 1.
3. If nums[index] < 0:
   - Duplicate found.
4. Else:
   - Mark visited by negating nums[index].
5. Return all duplicates.

## Complexity
Time: O(n)
Space: O(1) extra space

## Template Insight
Whenever:
- Values lie in [1, n].
- Array modification is allowed.
- Need presence/frequency information without extra space.

Think Index Marking.

## Common Mistakes
- Forgetting abs() after numbers become negative.
- Using value directly as index instead of value - 1.
- Missing the fact that array gets modified.
- Index out of bounds due to incorrect mapping.
