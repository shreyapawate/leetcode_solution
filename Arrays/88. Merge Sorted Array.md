class Solution {
    public int[] merge(int[] nums1, int m, int[] nums2, int n) {
        int ptr=m+n-1;
        int l=m-1;
        int r=n-1;
        while(l>=0 && r>=0){
            if(nums1[l]>nums2[r]){
                nums1[ptr]=nums1[l]; 
                l--;
                ptr--;
            }else{
                nums1[ptr]=nums2[r];   
                r--;
                ptr--;
            }
        }
        while(l>=0){
            nums1[ptr]=nums1[l];
            l--;
            ptr--;
        }
        while(r>=0){
            nums1[ptr]=nums2[r];
            r--;
            ptr--;
        }
        return nums1;  
    }
}
# 88. Merge Sorted Array

## Pattern
Two Pointers (Reverse Traversal)

## Recognition
- Two sorted arrays.
- Merge in-place.
- Extra space not preferred.
- nums1 has enough space to hold nums2.

## Core Idea
Start filling from the end.

Since the largest element among both arrays belongs at the last position, compare elements from the back and place the larger one at the end.

This avoids shifting elements.

## Algorithm
1. i = m - 1 (last valid element of nums1)
2. j = n - 1 (last element of nums2)
3. k = m + n - 1 (last position of nums1)
4. While i >= 0 and j >= 0:
   - Place larger element at k.
   - Move corresponding pointer.
5. Copy remaining elements of nums2 if any.

## Complexity
Time: O(m + n)
Space: O(1)

## Template Insight
Whenever:
- Merge sorted arrays in-place.
- Destination array has extra space.
- Need to avoid element shifting.

Think Reverse Two Pointers.

## Common Mistakes
- Merging from the front.
- Forgetting nums1 already has trailing zero placeholders.
- Copying remaining elements of nums1 (unnecessary).
- Incorrect initialization of k.

## Edge Cases
- m = 0
- n = 0
- All elements of nums2 smaller than nums1.
- All elements of nums2 larger than nums1.
