class Solution {
    public double myPow(double x, int n) {
        long binform=n;
        double ans=1;
        if(binform>0){
             while(binform>0){
            if(binform%2==1){
                ans=ans*x;  
            }
             x=x*x;
                binform=binform/2;
        }
           return ans;
        }else{
            binform=-binform;
            while(binform>0){
            if(binform%2==1){
                ans=ans*x;   
            }
             x=x*x;
                binform=binform/2;
        }
        return 1/ans;
        }
    }
}

# 50. Pow(x, n)

## Pattern
Binary Exponentiation (Fast Power)

## Recognition
- Calculate xⁿ.
- Exponent can be very large.
- Brute force O(n) will TLE.
- Need logarithmic solution.

## Core Idea
Use binary representation of exponent.

- If current bit is 1 → include current power in answer.
- Square the base after every step.
- Halve the exponent.

This reduces multiplications from n to log n.

## Algorithm
1. Convert n to long.
2. If n < 0:
   - x = 1/x
   - power = -power
3. While power > 0:
   - If odd → ans *= x
   - x *= x
   - power /= 2
4. Return ans.

## Complexity
Time: O(log n)
Space: O(1)

## Template Insight
Whenever:
- a^b
- Large exponent
- Modular exponentiation
- Repeated multiplication

Think Binary Exponentiation.

## Common Mistakes
- Using O(n) multiplication.
- Forgetting negative exponent case.
- Overflow on Integer.MIN_VALUE.
- Negating int directly instead of converting to long first.

## Edge Cases
- n = 0 → return 1
- x = 0
- n < 0
- n = Integer.MIN_VALUE
