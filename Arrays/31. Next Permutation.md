class Solution {
    public void nextPermutation(int[] nums) {
        int n =nums.length;
        int pivot=-1;
        for(int i=n-1;i>0;i--){
            if(nums[i]>nums[i-1]){
                pivot=i-1;
                break;
            }
        }
        if(pivot!=-1){
        for(int i=n-1;i>pivot;i--){
            if(nums[i]>nums[pivot]){
                int temp=nums[i];
                nums[i]=nums[pivot];
                nums[pivot]=temp;
                break;
            }
        }}
        int i=pivot+1;
        int j=n-1;
        while(i<j){
            int temp=nums[i];
                nums[i]=nums[j];
                nums[j]=temp;
                i++;
                j--;
        }
    }
}
# 31. Next Permutation

## Pattern
Greedy + Two Pointers

## Recognition
- Need the lexicographically next greater arrangement.
- Must modify array in-place.
- Asked for "next larger permutation".

## Core Idea
Find the first position from the right where increasing is possible.

1. Find pivot → first index `i` from right such that nums[i] < nums[i+1].
2. Find the smallest element greater than nums[i] on its right.
3. Swap them.
4. Reverse the suffix to get the smallest possible arrangement.

## Algorithm
1. Traverse from right to find pivot.
2. If pivot exists:
   - Find element just larger than pivot.
   - Swap.
3. Reverse entire suffix after pivot.
4. If no pivot exists, reverse whole array.

## Complexity
Time: O(n)
Space: O(1)

## Template Insight
Whenever:

Think:
Pivot → Swap → Reverse

## Common Mistakes
- Finding pivot from left instead of right.
- Sorting suffix instead of reversing it.
- Forgetting the case when array is completely decreasing.
- Choosing first greater element instead of rightmost greater element.
