# 74. Search a 2D Matrix

## Pattern
Binary Search

## Recognition
- Rows are sorted.
- First element of each row > last element of previous row.
- Entire matrix behaves like a sorted 1D array.
- Need O(log(m*n)) solution.

## Core Idea
Treat the matrix as a flattened sorted array.

Mapping:

row = mid / cols

col = mid % cols

Perform normal binary search on indices [0, m*n - 1].

## Algorithm
1. rows = matrix.length
2. cols = matrix[0].length
3. low = 0, high = rows * cols - 1
4. While low <= high:
   - mid = low + (high - low)/2
   - Convert mid → (row, col)
   - Compare matrix[row][col] with target
5. Return true if found, else false

## Complexity
Time: O(log(m*n))
Space: O(1)

## Template Insight
Whenever:
- Matrix is globally sorted.
- Row transition preserves ordering.
- Need efficient search.

Think:
Flatten Matrix → Binary Search

## Common Mistakes
- Running binary search on every row → O(m log n).
- Incorrect mapping:
  - row = mid / cols
  - col = mid % cols
- Forgetting matrix behaves like one sorted array.
- Integer overflow in mid calculation.
