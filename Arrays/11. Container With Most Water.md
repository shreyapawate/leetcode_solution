# 11. Container With Most Water

## Pattern
Two Pointers

## Recognition
- Need maximum area/value between two ends.
- Area depends on both width and minimum height.
- Brute force O(n²) is too slow.

## Core Idea
Area = min(height[left], height[right]) × (right - left)

The shorter line limits the water level.
Moving the taller line cannot increase the height limit, so always move the shorter pointer.

## Algorithm
1. left = 0, right = n - 1
2. Calculate current area.
3. Update maximum area.
4. Move the pointer with smaller height.
5. Repeat until pointers meet.

## Complexity
Time: O(n)
Space: O(1)

## Template Insight
Whenever:
- Need optimal pair from both ends.
- Width decreases as pointers move.
- Minimum/maximum of two values determines result.

Think Two Pointers from Opposite Ends.

## Common Mistakes
- Moving the taller pointer.
- Using max(height[left], height[right]).
- Trying all pairs (O(n²)).
- Wrong loop condition (`while(left < right)` is required).

## Key Observation
If height[left] < height[right]:
- Current area is limited by height[left].
- Keeping left fixed and moving right inward only decreases width.
- To potentially get a larger area, increase the limiting height by moving left.
