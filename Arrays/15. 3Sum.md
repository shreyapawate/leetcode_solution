# 15. 3Sum

## Pattern
Sorting + Two Pointers

## Recognition
- Find triplets.
- Sum equals target (usually 0).
- Need unique triplets.
- Brute force O(n³) is too slow.

## Core Idea
Fix one element and reduce the problem to Two Sum.

After sorting:
- Fix nums[i].
- Find two numbers in the remaining array whose sum is -nums[i].
- Use two pointers to avoid O(n²) extra space and handle duplicates efficiently.

## Algorithm
1. Sort array.
2. For each index i:
   - Skip duplicate values.
   - left = i + 1
   - right = n - 1
3. Calculate sum.
4. If sum == 0:
   - Store triplet.
   - Skip duplicates.
   - Move both pointers.
5. If sum < 0:
   - left++
6. If sum > 0:
   - right--

## Complexity
Time: O(n²)
Space: O(1) extra (excluding output)

## Template Insight
Whenever:
- Need unique triplets.
- Array can be sorted.
- Looking for k-sum style problems.

Think:
Sort → Fix One → Two Sum

## Common Mistakes
- Forgetting to sort.
- Not skipping duplicate i values.
- Not skipping duplicate left/right values after finding a triplet.
- Using HashSet unnecessarily.
- Moving only one pointer after finding a valid triplet.

## Key Observation
After sorting:
- Sum too small → increase left.
- Sum too large → decrease right.
- Sorted order lets us adjust sum in O(1).
