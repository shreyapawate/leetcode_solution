class Solution {
    public int[] productExceptSelf(int[] nums) {
        int[] arr= new int[nums.length];
        int[] pre= new int[nums.length];
        int[] suf= new int[nums.length];
        pre[0]=1;
        suf[nums.length-1]=1;
        //prefix product
        for(int i=1; i<nums.length;i++){
            pre[i]=pre[i-1]*nums[i-1];
            
            }

            for(int j =nums.length-2;j>=0;j--){
                suf[j]=suf[j+1]*nums[j+1];
            }

            for(int i =0;i<nums.length;i++){
                arr[i]=pre[i]*suf[i];
            }
            return arr;
        }
       
        
}
# 238. Product of Array Except Self

## Pattern
Prefix Product + Suffix Product

## Recognition
- Need product of all elements except current index.
- Division is not allowed.
- O(n) time expected.

## Core Idea
For index i:

productExceptSelf[i] =
(Product of elements on left) × (Product of elements on right)

Store left products in answer array.
Traverse from right while maintaining suffix product.

## Algorithm
1. ans[0] = 1.
2. Fill ans with prefix products.
3. Maintain suffix = 1.
4. Traverse from right:
   - ans[i] *= suffix
   - suffix *= nums[i]
5. Return ans.

## Complexity
Time: O(n)
Space: O(1) extra space
(Answer array is not counted.)

## Template Insight
Whenever:
- Need result excluding current index.
- Division is forbidden.
- Product/sum of left and right sides is required.

Think Prefix + Suffix.

## Common Mistakes
- Using division.
- Creating separate prefix and suffix arrays (works but uses O(n) extra space).
- Incorrect initialization:
  - prefix starts with 1
  - suffix starts with 1
- Off-by-one errors while building prefix/suffix.

## Revision (15 sec)
- ans stores prefix products.
- Traverse right with suffix variable.
- ans[i] = prefix × suffix.
- No division needed.
- O(n) time, O(1) extra space.
