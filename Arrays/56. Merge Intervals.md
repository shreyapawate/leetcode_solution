# 56. Merge Intervals

## Pattern
Sorting + Interval Merging

## Recognition
- Given intervals [start, end].
- Need to combine overlapping intervals.
- Asked for merged/non-overlapping result.

## Core Idea
Sort intervals by start time.

If current interval overlaps with the last merged interval:
- Extend the end time.

Otherwise:
- Start a new interval.

## Algorithm
1. Sort intervals by start value.
2. Add first interval to result.
3. Traverse remaining intervals:
   - If current.start <= lastMerged.end:
     - Merge by updating end.
   - Else:
     - Add current interval.
4. Return result.

## Complexity
Time: O(n log n)  (sorting)
Space: O(n)       (result list)

## Template Insight
Whenever:
- Overlapping ranges.
- Meeting schedules.
- Time intervals.
- Range consolidation.

Think:
Sort → Compare with Last Merged

## Common Mistakes
- Forgetting to sort first.
- Comparing with previous interval instead of last merged interval.
- Using `<` instead of `<=` for overlap check.
- Updating start instead of end during merge.
