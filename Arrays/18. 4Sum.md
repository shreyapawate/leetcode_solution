# 18. 4Sum

## Pattern
Sorting + Two Pointers + K-Sum

## Recognition
- Find unique quadruplets.
- Sum equals target.
- Duplicate handling is important.
- Extension of 3Sum.

## Core Idea
Fix two elements and solve the remaining problem using Two Pointers.

After sorting:
- Fix i.
- Fix j.
- Use left and right pointers to find the remaining two numbers.

3Sum:
Fix 1 + Two Pointers

4Sum:
Fix 2 + Two Pointers

## Algorithm
1. Sort array.
2. Loop i from 0 to n-1:
   - Skip duplicates.
3. Loop j from i+1 to n-1:
   - Skip duplicates.
4. left = j + 1, right = n - 1
5. Calculate sum of four numbers.
6. If sum == target:
   - Store quadruplet.
   - Skip duplicates.
   - Move both pointers.
7. If sum < target:
   - left++
8. If sum > target:
   - right--

## Complexity
Time: O(n³)
Space: O(1) extra (excluding output)

## Template Insight
K-Sum Pattern:

- 2Sum → O(n)
- 3Sum → O(n²)
- 4Sum → O(n³)

General idea:
Sort → Fix elements → Reduce to Two Sum.

## Common Mistakes
- Not sorting first.
- Forgetting duplicate checks for i and j.
- Forgetting duplicate checks after finding a valid quadruplet.
- Integer overflow while calculating sum.
- Using int instead of long for sum.

## Edge Cases
- Large values causing overflow.
- Multiple duplicate numbers.
- Negative target.
- Less than 4 elements.
