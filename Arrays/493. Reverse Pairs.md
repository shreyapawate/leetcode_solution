# 493. Reverse Pairs

## Pattern
Merge Sort + Two Pointers

## Recognition
- Count pairs `(i, j)` where `i < j`.
- Condition involves elements from left and right halves.
- Brute force O(n²) will TLE.
- Asked to count while sorting.

## Core Idea
During Merge Sort:

- Left half and right half are already sorted.
- Before merging, count valid reverse pairs using two pointers.
- Since both halves are sorted, the right pointer never moves backward.

Condition:

```text
nums[i] > 2 * nums[j]
```

## Algorithm
1. Divide array using Merge Sort.
2. Recursively sort left and right halves.
3. Before merging:
   - Count reverse pairs using two pointers.
4. Merge the two sorted halves.
5. Return total count.

## Pseudocode

```java
mergeSort(low, high):

    if(low >= high)
        return 0

    mid = (low + high) / 2

    count =
        mergeSort(low, mid) +
        mergeSort(mid+1, high)

    j = mid + 1

    for(i = low to mid):

        while(j <= high && nums[i] > 2L * nums[j])
            j++

        count += (j - (mid + 1))

    merge(low, mid, high)

    return count
```

## Key Syntax

```java
int mid = low + (high - low) / 2;

while(j <= high && nums[i] > 2L * nums[j])

count += j - (mid + 1);

2L * nums[j]     // prevent integer overflow

merge(low, mid, high);
```

## Complexity

Time: O(n log n)

Space: O(n)

## Template Insight

Whenever:
- Count pairs with `i < j`.
- Pair condition compares left and right halves.
- Efficient counting while sorting.

Think:
