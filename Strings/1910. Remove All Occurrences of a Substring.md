# 1910. Remove All Occurrences of a Substring

## Pattern
String Simulation + Stack (StringBuilder)

## Recognition
- Repeatedly remove a substring.
- Removal may create new occurrences.
- Need efficient string processing.

## Core Idea
Treat `StringBuilder` as a stack.

- Append characters one by one.
- After each append, check if the last `part.length()` characters equal `part`.
- If yes, delete that suffix immediately.

This naturally handles newly formed occurrences.

## Algorithm
1. Create an empty `StringBuilder`.
2. Traverse each character of `s`:
   - Append the character.
   - If builder length ≥ part length:
     - Compare the suffix with `part`.
     - If equal, delete that suffix.
3. Return the final string.

## Pseudocode

```java
sb = new StringBuilder()

m = part.length()

for(ch : s):

    sb.append(ch)

    if(sb.length() >= m):

        if(last m characters == part):

            sb.delete(sb.length() - m, sb.length())

return sb.toString()
```

## Key Syntax

```java
StringBuilder sb = new StringBuilder();

sb.append(ch);

sb.length();

sb.substring(sb.length() - m).equals(part);

sb.delete(sb.length() - m, sb.length());

sb.toString();
```

## Complexity

Time: O(n × m)
- `n` = length of `s`
- `m` = length of `part`

Space: O(n)

> For the given constraints (`n ≤ 1000`), this is efficient and accepted.

## Template Insight

Whenever:
- Characters are processed sequentially.
- Need to repeatedly remove the most recently formed pattern.
- Operations occur at the end of the string.

Think:
