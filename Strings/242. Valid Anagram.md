# 242. Valid Anagram

## Pattern
Frequency Counting (Hashing)

## Recognition
- Check if two strings contain the same characters.
- Character frequencies matter.
- Order does not matter.
- Strings contain lowercase English letters.

## Core Idea
Count the frequency of each character in one string and subtract the frequencies using the other string.

- If all frequencies become `0`, the strings are anagrams.

> Since the characters are lowercase English letters (`a-z`), use a fixed-size frequency array instead of a HashMap.

## Algorithm
1. If lengths differ, return `false`.
2. Create a frequency array of size `26`.
3. Traverse both strings simultaneously:
   - Increment frequency for `s`.
   - Decrement frequency for `t`.
4. Check if every frequency is `0`.
5. Return `true` if all are zero.

## Pseudocode

```java
if(s.length() != t.length())
    return false

freq[26] = {0}

for(i = 0 to n-1):

    freq[s.charAt(i) - 'a']++

    freq[t.charAt(i) - 'a']--

for(count : freq):

    if(count != 0)
        return false

return true
```

## Key Syntax

```java
int[] freq = new int[26];

freq[s.charAt(i) - 'a']++;

freq[t.charAt(i) - 'a']--;

for(int count : freq)

s.length();
```

## Complexity

Time: O(n)

Space: O(1)

## Template Insight

Whenever:
- Compare character frequencies.
- Characters belong to a small fixed range (e.g., `a-z`).

Think:
**Frequency Array > HashMap**

Use a `HashMap` only when the character set is not fixed (e.g., full Unicode).
